### EvidencePort.publish_raw payload contract (P4 must normalize to this)

P1 calls `publish_raw(event_type: str, payload: dict) -> correlation_id`, then
`wait_for_evidence(correlation_id) -> Evidence`, then
`query(run_id, *, claim=None, kind=None) -> list[Evidence]`. The payload P1
sends is Evidence-shaped (minus `evidence_id`):

```json
{"run_id","candidate_id","kind","claim","value","source","source_ref","provenance","policy_decision","occurred_at"}
```

Diagnosis treats evidence of `kind in {"enrichment","tool"}` as an obtained claim
and `kind == "call"` as a call record; live diagnosis reads ONLY normalized
evidence from `query()`, never raw transcripts.

### Generated-tool lifecycle (runtime agent authors these — P1 never commits them)

`generated_tools/fact_b_tool.py`, `generated_tools/fact_b_tool.manifest.json`,
`generated_tools/test_fact_b_tool.py`. Authoring is path-restricted (any change
outside these three files is rejected via `git status --porcelain`), conformance
gets exactly one automated repair attempt, then the tool is merged through P3's
`RepoPort` and loaded by capability via `ToolRegistry.reload()` — the orchestrator
never imports the generated module directly.
